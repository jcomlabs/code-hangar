## Status vocabulary

- `PASS`: direct current evidence proves the requirement.
- `FAIL`: current evidence contradicts the requirement.
- `PENDING`: the required test has not yet run.
- `BLOCKED`: the test cannot run until the recorded infrastructure condition changes.
- `N/A`: reviewed and proven outside the build's intended capability.

## Evidence runner

Run one or more explicit lanes; every invocation creates a timestamped manifest
and individual logs under `.local/acceptance/v0.1.1/`:

```powershell
pwsh scripts/acceptance-v011.ps1 -Lane Baseline
pwsh scripts/acceptance-v011.ps1 -Lane RealData
pwsh scripts/acceptance-v011.ps1 -Lane DataStress
pwsh scripts/acceptance-v011.ps1 -Lane WslOff
pwsh scripts/acceptance-v011.ps1 -Lane RuntimePerf
pwsh scripts/acceptance-v011.ps1 -Lane McpHarness
pwsh scripts/acceptance-v011.ps1 -Lane Gate3
pwsh scripts/mcp-claude-real.ps1 # requires prior manual `claude auth login`
```

`DataStress` creates all of its trees, ACLs, junctions and transcripts under
temporary directories. It restores the temporary deny ACL before cleanup and
records the run in `.local/acceptance/v0.1.1/`; the current passing manifest is
`20260712-051256/manifest.json`.

`RuntimePerf` rebuilds the exact strict-core release executable and gives every run a
new disposable `APPDATA`, `LOCALAPPDATA`, home and Codex home below its evidence
directory. The startup figure is deliberately named *time to responsive main
window*: it does not claim that all asynchronous dashboard data is hydrated at
that instant. Memory covers the complete process tree, including WebView2, after
a fixed 1.5-second settle delay. The current evidence is
`20260712-104331/runtime-performance.json`.

The clean lifecycle is orchestrated by `scripts/sandbox-lifecycle.ps1` in a
network-disabled Windows Sandbox. It stages both 0.1.1 installers, the 0.1.0
baseline, a test-only catalog helper and a deterministic project fixture. The
authoritative exact-final run is
`sandbox-lifecycle/20260712-110835/lifecycle-manifest.json`; it contains 25 fresh
required command results (IDs through 26; obsolete retry 07 is absent), zero ignored
attempts and a stopped guest agent. `-ValidateOnly -EvidenceDir <run>` revalidates
persisted evidence without relaunching the VM, while `-Resume -EvidenceDir <run>`
